# クロマのマジカルチェイン 盤面回転式パズル実装計画 v1

> この文書は初期案として保存する。現在の横長UIと最新ルールを反映した実装基準は `ROTATION_PUZZLE_IMPLEMENTATION_PLAN_V2.md` を参照する。

## 1. 文書の目的と位置付け

- 現行の落下式「クロマのマジカルチェイン」を直ちに置き換えず、盤面回転式パズルを別URLで試作するための実装計画を定義する。
- 横長UI改造、クロマ・敵キャラクター枠、固定対戦背景、HUD、BGM・SE、特殊技演出は可能な限り再利用する。
- 先に `LANDSCAPE_UI_IMPLEMENTATION_PLAN_V2.md` の残りタスクを進め、その後に本計画へ着手する。
- 試作が成立するまで、公開版 `/play/colorChain/` と既存テスト版 `/test/color-chain-mascot/` のゲームルールを変更しない。

## 2. 確定した方針

| 項目 | 採用方針 |
| --- | --- |
| 試作URL | `/test/color-chain-rotate/` |
| 公開状態 | ポータル一覧・サイトマップには載せず、URLを知っている人だけがアクセスする試験ページ |
| 基本操作 | 盤面内の交点をタップまたは左右スワイプし、周囲の2×2ブロックを回転 |
| 時計回り | 交点をタップ、または交点から右へスワイプ |
| 反時計回り | 交点から左へスワイプ |
| 長押し | 使用しない。ブラウザの長押しメニューとゲーム操作を競合させない |
| 不成立手 | 回転後にマジカルチェインが成立しなければ、短い演出後に元の配置へ戻す |
| 消去条件 | 同色が縦・横・斜めに4個以上つながった場合 |
| 補充 | 消去した列の上部ブロックを落とし、空いた上端から新しいブロックを補充 |
| 連鎖 | 落下・補充後に再び条件を満たした場合、Wチェイン以降として継続 |
| 特殊ブロック | 大消去の報酬に加え、補充時にも低確率で出現 |
| ステージクリア | 敵戦は封印ゲージ満タン。敵なしステージは消去数または指定ターゲット達成 |
| ゲームオーバー | 制限時間切れ |
| 敵の妨害 | 落下式前提の技を、2×2回転・交点・時間制ステージに合わせて再設計 |

## 3. 初期試作の推奨パラメータ

以下は実装開始時の仮値とし、Redmi Note 11を含む実機試験後に調整する。

- 盤面: 8列×8行。
- 通常色: 4色。高難度ステージだけ5色を候補とする。
- 制限時間: 90秒。
- タップ許容移動量: 10px前後。
- スワイプ成立距離: 24px前後、またはセル幅の45〜60%の小さい方。
- 回転アニメーション: 140〜190ms。
- 不成立時の停止: 80〜120ms表示してから逆回転で復元。
- 通常特殊ブロックの補充抽選: 新規補充1個につき合計2〜3%を上限の初期値とする。
- プリズムブレイク: 特殊ブロック抽選内のレア枠とする。
- 盤面上のランダム出現特殊ブロック数には上限を設ける。

8×8は確定値ではないが、スマートフォンでマスを大きく保ち、横長画面でクロマ・敵・HUDを同時表示する最初の検証値として最も扱いやすい。

## 4. 1手の進行

1. 入力可能状態で、プレイヤーが盤面内の交点をタップまたは左右スワイプする。
2. 交点を中心とする2×2範囲を発光枠で示す。
3. 指定方向へ4ブロックを回転する。
4. 盤面全体の縦・横・斜め一致を判定する。
5. 一致がなければ、盤面を元へ戻して入力可能状態へ戻る。
6. 一致があれば、その操作をマジカルチェイン1段階目として確定する。
7. 一致ブロックと誘発された特殊ブロックを消去する。
8. 各列へ重力を適用し、上端から不足数を補充する。
9. 新たな一致があればチェイン段階を1つ増やし、7へ戻る。
10. 一致がなくなったら封印ゲージ、ステージ目標、手詰まりを判定する。
11. クリア条件未達かつ時間が残っていれば、次の入力を受け付ける。

### 制限時間の扱い

- タイマーは原則としてプレイヤーが入力可能な間だけ減らす。
- 回転、消去、特殊技、落下、補充、大技演出など、プレイヤーが操作できない解決処理中はタイマーを停止する。
- 残り時間が0になった瞬間に成立済みのチェインがある場合は、最後のチェイン解決まで完了してから勝敗を判定する。
- 最後の解決でクリア条件を満たした場合はステージクリアを優先する。
- ポーズ、説明、設定画面を開いている間はタイマーを停止する。

これにより、大連鎖や大技演出がプレイヤーの残り時間を奪わず、成功するほど不利になる問題を防ぐ。

## 5. 交点タップ・スワイプ操作仕様

### 交点の定義

- 8×8盤面では、4マスに囲まれた内側の交点が7×7個存在する。
- 各交点は周囲の2×2範囲へ一意に対応する。
- 判定領域は見た目の点より広く取るが、隣接交点の領域とは重ならないようセル寸法から算出する。
- 初回チュートリアルでは交点を淡く表示し、通常時はホバー、タッチ開始、操作候補時だけ強く表示する。

### 入力判定

- 交点付近で押下し、移動量が閾値未満なら時計回りタップとする。
- 交点付近から水平方向へ右スワイプすると時計回りとする。
- 交点付近から水平方向へ左スワイプすると反時計回りとする。
- 縦方向の移動が大きい場合は回転を発動せず、誤操作として安全にキャンセルする。
- 回転開始後は `pointerId` を固定し、別の指やマウス入力を混ぜない。
- `pointercancel`、画面外移動、タブ非表示、ポーズ時に選択状態を解除する。
- 盤面だけに `touch-action: none` を設定し、盤面外の通常スクロールやブラウザ操作へ影響を広げない。
- 回転・解決処理中の追加入力はキューへ積まず、無視する。

### PC・アクセシビリティ

- 左クリックまたはEnterで時計回り。
- 左右ドラッグ、またはキーボードの左右操作で回転方向を指定できる代替入力を用意する。
- 選択交点、対象2×2、回転方向を色だけでなく枠、矢印、ARIAラベルでも伝える。

## 6. 盤面生成と手詰まり対策

### 初期盤面

- 初期状態で縦・横・斜め4個以上が完成していない盤面を生成する。
- 時計回り・反時計回りを含め、最低1つ以上の成立手があることを確認する。
- 条件を満たさない場合は盤面を作り直す。試行回数に上限を設け、上限到達時は確実に成立するテンプレートを混ぜる。

### 手詰まり

- 安定盤面になった時点で、すべての内側交点を時計回り・反時計回りに仮回転し、成立手を列挙する。
- 成立手が0件なら、短い「彩鎖を結び直しています」演出の後に自動シャッフルする。
- シャッフル後も初期盤面と同じく「完成済み一致なし」「成立手1件以上」を保証する。
- ヒント機能は成立手一覧から1件を選び、一定時間無操作の場合だけ交点を発光させる。

## 7. ロジック構成

落下式のコードへ直接条件分岐を増やさず、回転式専用の純粋ロジックを新設する。

想定ファイル:

- `src/games/colorChain/rotationLogic.ts`
- `src/games/colorChain/rotationGesture.ts`
- `src/games/colorChain/ColorChainRotationTest.tsx`
- `scripts/test-color-chain-rotation.mjs`

主要な純粋関数:

- `createRotationBoard()`
- `rotateSquare(board, row, column, direction)`
- `findRotationMatches(board)`
- `isProductiveRotation(board, row, column, direction)`
- `enumerateProductiveRotations(board)`
- `collapseColumns(board)`
- `refillBoard(board, randomSource, settings)`
- `resolveRotationChain(board, settings)`
- `createPlayableBoard(settings)`
- `shuffleToPlayableBoard(board, settings)`

既存 `logic.ts` のうち、色トークン、縦・横・斜め一致、特殊ブロック誘発、消去、重力、スコア倍率は、落下ペアへ依存しない範囲を共通モジュールへ切り出して再利用する。既存公開版の挙動を変える大規模な移動は、回帰テストを追加してから行う。

進行状態は最低限、次の状態へ分ける。

```text
idle → ready → rotating → validating
                    ├─ no-match → reverting → ready
                    └─ matched  → clearing → falling → refilling
                                                 ├─ matched → clearing
                                                 └─ stable  → goal-check → ready / clear / timeout
```

## 8. 特殊ブロック

### 維持する基本ルール

- チェインボム: 周囲3×3を消去。
- チェインピラー: 縦1列を消去。
- チェインウェーブ: 横1列を消去。
- プリズムブレイク: 対象色を全消去。
- 同種特殊ブロック隣接時のグランドチェインボム、トリニティピラー、トリニティウェーブ、プリズムノヴァ。
- 縦・横・斜め5個以上や複数方向同時消去による報酬特殊ブロック生成。

### 回転式での扱い

- 特殊ブロックも通常ブロックと同様に2×2回転で移動する。
- 特殊ブロックそのものは通常色一致の本数に含めない。
- 隣接色のマジカルチェイン、別の特殊技への巻き込み、同種特殊ブロック隣接で発動する。
- 補充抽選で特殊ブロックが出現しても、着地しただけでは即時発動しない。
- ランダム補充と大消去報酬が同時に増えすぎないよう、盤面上限、連続非出現救済、レア枠率を難易度設定へ分離する。
- 自動補充だけでスーパー技が成立する偶発性は残すが、頻発する場合は同種特殊ブロック同士が直接隣接しにくい補充制御を追加する。

## 9. ステージ目標と勝敗

### 敵がいるステージ

- マジカルチェイン、連鎖、大技で敵の封印ゲージを増やす。
- 制限時間内に封印ゲージが100%になればクリア。
- 時間切れ時に100%未満ならゲームオーバー。
- 通常消去、連鎖段階、特殊技の強度を封印値へ変換し、大技ほど大きく増える。

### 敵がいないステージ

ステージ定義で次のいずれかを選ぶ。

- 制限時間内に指定数のブロックを消去する。
- 指定色を必要数消去する。
- 指定特殊ブロックを必要回数発動する。
- 指定チェイン数を達成する。
- ターゲットブロックをすべて消去する。

複数条件のAND／ORを初回から汎用化しすぎず、最初は「総消去数」と「封印ゲージ」の2種類で試作する。

## 10. 敵妨害技の回転式への再設計方針

すべての妨害は「予告 → 発動 → 短時間または解除条件 → 復帰」を守り、全交点を同時に封鎖しない。

| 敵 | 回転式での妨害候補 |
| --- | --- |
| モコスライム | 1〜2個の交点へぬめりを付け、一時的に使用不可または回転復帰を少し遅くする。初期敵なので盤面情報は隠さない |
| チェインバット | ムーンミストで下1〜2行を半透明にする。色記号と輪郭は残し、短時間だけ使用する。大技のみ対象行を増やす |
| フロストン | 予告した交点を氷結し、数秒または隣接チェイン成立まで回転不可にする。タイマーそのものを直接大量に減らさない |
| ポンポンゴースト | 一部ブロックを半透明化または一時的に位置だけ見せる。記号を完全には消さず、推測可能性を残す |
| キノコノコ | 胞子で一部ブロックの外観を似せるが、色覚補助記号は維持する。特殊ブロックの機能自体は奪わない |
| ミラ | 複数交点への鎖封印、回転方向予告の撹乱、短時間の領域封鎖を組み合わせる。ボス戦でも必ず成立手を1つ以上残す |

妨害後に成立手が0件になった場合は、敵技を解除または盤面を補正し、プレイヤーが操作不能のまま時間切れになる状態を防ぐ。敵行動ゲージは時間または成功手数で増やし、マジカルチェインと特殊技で減速・中断できる余地を残す。

## 11. 横長UIとの接続

- 現在の「クロマ／中央盤面／敵」の3列シェルを維持する。
- 盤面が8×16から8×8候補になるため、中央マスを大きくでき、上下に目標、タイマー、CHAIN表示を置く余裕が増える。
- NEXTとHOLDは回転式では不要になるため、その領域を制限時間、封印ゲージ、ステージ目標、ヒントへ置き換える。
- 落下・左右移動・回転ボタンは撤去し、盤面交点操作、ポーズ、ヒント、補助技を主要操作にする。
- 説明・設定は `LANDSCAPE_UI_IMPLEMENTATION_PLAN_V2.md` のP1-Bどおり別画面化する。
- 宝石風ブロック、CHAINテロップ、クロマ演出、固定背景、BGM・SEは継続利用する。
- 通常チェインSE、大技SE、ゲームオーバーSEに加え、将来は回転成功、不成立復帰、補充、ステージクリアの短いSEを追加候補とする。

横長UIの残作業は進めてよいが、8×16盤面、落下操作ボタン、NEXT、HOLDへ依存するピクセル単位の最終調整は、回転式試作後に再評価する。

## 12. 実装フェーズ

### Phase R0: 仕様固定とテスト境界

- 本文書の仮パラメータを確定する。
- 新URL、内部ID、ランキングID、localStorageキーを既存版と分離する。
- 乱数を注入可能にし、盤面生成と補充を再現可能にする。

### Phase R1: 純粋ロジック試作

- 8×8盤面生成、2×2回転、不成立復帰判定を実装する。
- 縦・横・斜め一致、消去、重力、補充、連鎖を実装する。
- 初期一致なし、成立手保証、手詰まり判定、自動シャッフルをテストする。
- この段階ではキャラクターと特殊ブロックを接続しない。

### Phase R2: 入力と最小UI

- `/test/color-chain-rotate/` を追加する。
- 交点タップ、左右スワイプ、マウス、キーボード代替操作を接続する。
- 対象2×2、回転方向、不成立復帰、入力ロックを視覚化する。
- Redmi Note 11で交点判定とスワイプ閾値を確認する。

### Phase R3: 連鎖演出と音

- CHAINテロップ、鎖エフェクト、スコア、BGM、通常チェインSEを接続する。
- 自動連鎖中はタイマーを止める。
- ステージクリア、時間切れ、最終チェイン優先判定を実装する。

### Phase R4: 特殊ブロック移植

- 各特殊ブロックとスーパー効果を1種類ずつ移植する。
- 補充抽選、大消去報酬、盤面上限、レア率を追加する。
- 特殊技SE、クロマの大技反応、封印ゲージ加算を接続する。

### Phase R5: 敵と妨害技

- 最初にモコスライムを接続する。
- 敵行動ゲージ、妨害予告、交点ぬめり、被弾、敗北を実装する。
- 敵技適用後も成立手が残ることを自動検証する。
- 他の敵は1体ずつ追加し、複数妨害の同時実装を避ける。

### Phase R6: 比較評価と移行判断

- 落下式テスト版と回転式テスト版を並行稼働する。
- 操作ミス、平均チェイン、プレイ時間、特殊技発動頻度、スマートフォンのスクロール有無を比較する。
- 回転式を正式採用する場合のみ、公開版の名称、説明、ランキング、記録移行方針を決める。
- 旧落下式はコードを即削除せず、移行完了まで復旧可能な状態を保つ。

## 13. QA・完了条件

### ロジック

- 2×2回転で4ブロックが正しい位置へ移る。
- 時計回り4回、反時計回り4回で元の盤面へ戻る。
- 不成立手はブロック内容・特殊状態を完全に復元する。
- 初期盤面に完成済み一致がない。
- 初期盤面とシャッフル後に成立手が1件以上ある。
- 縦・横・斜め、同時複数方向、5個以上、交差消去を正しく検出する。
- 補充後の連鎖が段階順に解決し、無限処理にならない。
- 特殊ブロックと報酬ブロックの生成位置が消去対象と競合しない。

### 入力・表示

- 交点タップと左右スワイプを誤判定しにくい。
- 縦スワイプ、複数指、pointer cancelで盤面が壊れない。
- 回転・解決中に二重入力できない。
- PC、Android、iOS、縦横画面で盤面全体を表示できる。
- 色、記号、特殊ブロック、交点、妨害状態を識別できる。
- 日本語・英語・視覚演出軽減・サウンドOFFで進行情報が欠けない。

### 勝敗

- タイマーが入力可能時だけ減少する。
- ポーズ、設定、解決演出中は減少しない。
- 時間切れ直前の成立チェインを最後まで解決する。
- 最終チェインで目標達成した場合はクリアを優先する。
- ゲージ、総消去数、ターゲット条件をステージ定義から切り替えられる。

## 14. ユーザー側で後に必要な確認

初期ロジック試作に追加素材は不要。既存のブロック、クロマ、BGM、SE、CSS背景で開始できる。

実機試作時に必要な確認:

1. Redmi Note 11で交点タップが狭すぎないか。
2. 右スワイプと左スワイプが意図した方向へ感じられるか。
3. 不成立復帰が遅すぎず、失敗理由が理解できるか。
4. 8×8、4色、90秒が簡単すぎる／忙しすぎる状態になっていないか。
5. 補充特殊ブロックの頻度と偶発連鎖が爽快感を高めているか。
6. 通常SEと大技SEの音量、連続再生時の聞き疲れがないか。

## 15. 着手前に最終決定する仮項目

- 盤面を8×8で開始するか。
- 初期制限時間を90秒とするか。
- 封印ゲージの初期必要量。
- 不成立復帰アニメーションの長さ。
- 交点判定半径とスワイプ成立距離。
- 特殊ブロックの補充率、盤面上限、救済抽選間隔。
- 敵なし試作の初期目標を総消去数とするか。

上記以外の基本方針は本書の内容で固定し、横長UI改造の残作業後にPhase R0から着手する。

# ミラ 戦闘状態とモーション対応表 v1

## 文書の位置付け

- 状態: Phase.1 構成案
- 対象: 将来のミラ戦、および先行テスト用の敵キャラクターパネル
- 目的: ゲームイベント、ミラの表情・全身モーション、画面効果、台詞を一つの状態管理へ接続する
- 現在の `カラーチェイン` 本編へ、敵側ゲージや妨害を直ちに追加するものではない
- 決定事項: ミラ戦の進行値は、内部実装名にかかわらず画面上では **封印ゲージ** と表示する
- 決定事項: ボイスは当面実装しない。画面上の台詞・説明文は日本語と英語を同時制作する

## 基本方針

- ミラの演出中も盤面操作を止めない
- 通常チェインごとに大きく反応させず、重要な連鎖・特殊技・妨害だけを目立たせる
- 同時に複数イベントが起きた場合は、優先度の高い状態だけを再生する
- 一度限りの動作を終えたら、`警戒待機` または `通常待機` へ戻す
- `prefers-reduced-motion` では移動・揺れ・点滅を減らし、静止表情の切替だけを残す
- ボイス、画面効果、キャラクター表示は個別にOFFへできる設計とする

## 状態の優先順位

優先度は数値が高いほど強い。高優先度の動作中は、低優先度のイベントで上書きしない。

| 優先度 | 区分 | 例 |
| --- | --- | --- |
| 100 | 戦闘終了 | 勝利、敗北、撤退 |
| 90 | 大ダメージ・気絶 | 4連鎖以上、スーパー特殊技、妨害中断 |
| 80 | 大技発動 | ブロークン・プリズン |
| 70 | 通常妨害発動 | ノワールチェイン、ミラージュ・ロック |
| 60 | 通常被弾 | 2〜3連鎖、通常特殊技 |
| 50 | 妨害予告 | 詠唱、鏡片の集束 |
| 30 | 挑発・成功反応 | 妨害成功、プレイヤーのミス |
| 10 | 待機差分 | 杖回し、帽子直し、腕組み |
| 0 | 通常待機 | 呼吸、瞬き、軽い上下動 |

## 現在のゲームイベントとの対応

現行のテスト版には、`idle`、チェイン数、特殊技、盤面危険、ゲームオーバーが既にある。ミラ戦の試作では、まず次のイベントだけを流用できる。

| 現在取得できるイベント | ミラの仮反応 |
| --- | --- |
| 長時間イベントなし | `idle-fidget` |
| Wチェイン | `hit-light` |
| トリプルチェイン | `hit-medium` |
| ハイパー以上 | `hit-heavy` → `stunned` → `recover` |
| チェインボム、チェインピラー、チェインウェーブ | `hit-light` |
| グランドチェインボム、トリニティピラー、トリニティウェーブ、プリズムブレイク | `hit-medium` |
| プリズムノヴァ | `hit-heavy` → `stunned` → `recover` |
| 盤面が最上部付近 | `taunt` を低確率で1回だけ再生 |
| プレイヤーのゲームオーバー | `victory` |

封印ゲージ、妨害ゲージ、予告、妨害成功・失敗は、ステージ制の戦闘コントローラーで新規追加する。

## 状態遷移ルール

1. `victory`、`defeat`、`retreat` 中は他状態へ遷移しない
2. `hit-heavy` は詠唱中の妨害を中断できる
3. `hit-light` は大技詠唱を中断せず、短い表情差分だけ重ねる
4. 同じ被弾が短時間に連続した場合、最初の1回だけ全身モーションを再生する
5. `idle-fidget` と `taunt` は、前回再生から最低8秒空ける
6. タブ非表示、ポーズ、キャラクター非表示中はタイマーを進めない
7. モーション終了後は、封印ゲージの残りが少なければ `guard-idle`、それ以外は `idle` へ戻す

## 実装用素材の推奨最小構成

設定画は完成しているが、Web実装には個別の背景透過素材が必要になる。

| 優先度 | 素材 | 最小枚数 |
| --- | --- | --- |
| 必須 | 通常待機＋瞬き | 2 |
| 必須 | 挑発 | 1 |
| 必須 | 3種の妨害魔法 | 3 |
| 必須 | 軽・中・大被弾 | 3 |
| 必須 | 気絶＋回復 | 2 |
| 必須 | 勝利＋敗北＋撤退 | 3 |
| 任意 | 帽子直し＋杖回し | 2 |
| 任意 | 各動作の開始・戻り中間フレーム | 動作ごとに1〜2 |

初回テストは静止画切替とCSS移動で構成し、必要性を確認してから中間フレームを増やす。

## ユーザー側で必要な判断・素材

現時点で、新しいイラストや外部サービスの登録は不要。実装へ進む前に次を決める。

1. 生成済み設定画のうち、衣装・杖・顔立ちの最終基準にする1枚を承認する
2. W／トリプル／ハイパー以上に対する被弾強度が適切か確認する
3. 画面表示名は `封印ゲージ` とする。内部変数名は実装時に決め、表示文言と切り離す
4. 当面はボイスを使わず、イラスト、モーション、画面効果、日英テキストで演出する
5. ボイスを追加する場合は、技名と勝利・敗北時の重要な短い台詞に限定する。英語版は日本語ボイスと英語テロップの組み合わせを基本案とする
6. ミラの台詞、説明文、状態表示は日本語と英語を同時制作する
7. テスト版で、演出頻度・妨害の強さ・画面占有率を実際にプレイして確認する

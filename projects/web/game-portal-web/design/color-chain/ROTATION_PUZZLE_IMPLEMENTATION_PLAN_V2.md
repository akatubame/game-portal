# クロマのマジカルチェイン 盤面回転式パズル詳細実装計画 v2

## 1. 文書の目的

- 現行の落下式「クロマのマジカルチェイン」を維持したまま、盤面回転式を別URLで試作する。
- 以前の `ROTATION_PUZZLE_IMPLEMENTATION_PLAN.md` を、現在の横長UI、特殊ブロック、敵妨害、背景・配色、最新の報酬条件に合わせて具体化する。
- 実装順、責務分割、状態遷移、テスト条件、実機確認項目を先に固定し、途中段階でも常にプレイ可能な状態を保つ。
- 本文書の承認まではゲームコードを変更しない。

### 2026-07-24 追加方針

- 盤面回転式は、将来のスマートフォンアプリ化を主目的とする。
- 先にブラウザ版でゲームロジック、操作感、演出を完成させ、自動テストとエージェントによる検証を続ける。
- ブラウザ版もレスポンシブな再配置を行わず、横長の固定論理画面を端末へ等比縮小して表示する。
- 最終段階で同じReact／TypeScriptのゲーム本体をスマートフォンのWebViewコンテナへ収め、ネイティブアプリ化する。
- ブラウザ版とアプリ版で盤面ルールを分岐させず、画面外周、画面方向、ストレージ、音声開始などの環境差だけをアダプター層へ分離する。

## 2. 現行実装を精査した結果

### 2.1 再利用できるもの

- 木製フレーム、暖色パネル、宝石と鎖の固定背景。
- 通常ブロック6色と4種の特殊ブロックの外観。
- クロマとモコスライムの透過素材、待機・被弾・危険・敗北・大技モーション。
- 通常チェインSE、大技SE、ゲームオーバーSE、ステージBGM。
- 説明・設定ダイアログ、日英切替、サウンド・画面効果・クロマ表示設定。
- CHAINテロップ、特殊技名テロップ、鎖エフェクト。
- 特殊ブロックの基本仕様とスーパー効果の考え方。
- ランキング、ベスト記録、PWA、Google Analytics、Cloudflare PagesのSPA配信基盤。

### 2.2 そのまま再利用しないもの

- `logic.ts` は8列×18行（隠し2行＋表示16行）を前提にしており、盤面探索・範囲判定・重力処理が固定定数を参照する。
- `ColorChain.tsx` は落下ペア、NEXT、HOLD、落下速度、着地、ゲームオーバー判定を含む大きなコンポーネントである。
- 現在のジェスチャー処理は、横移動・ソフトドロップ・ハードドロップを前提にしている。
- モコスライムの「次の着地でブロックを横へ滑らせる」妨害は、常時埋まっている8×8盤面では成立しない。
- スロータイムは落下速度を下げる技であり、時間制の回転式では効果を再定義する必要がある。

### 2.3 結論

- 回転式専用ロジックと専用Reactコンポーネントを新設する。
- 現行落下式へ大量の `if (rotationMode)` を追加しない。
- 色トークンなど小さく安全な定義だけを共有し、盤面解決処理は回転式用に8×8対応で実装する。
- 現行版の大規模リファクタリングは、回転式の成立確認後まで行わない。

## 5. 1手の進行と状態遷移

```text
idle
  ↓ start
ready
  ↓ pointer / keyboard
selecting → rotating → validating
                         ├─ 不成立 → reverting → ready / timeout
                         └─ 成立   → clearing
                                      ↓
                                  special-resolving
                                      ↓
                                   falling
                                      ↓
                                  refilling
                                      ├─ 再一致／スーパー成立 → clearing
                                      └─ 安定 → goal-check
                                                   ├─ clear
                                                   ├─ timeout
                                                   ├─ no-move → shuffling → ready
                                                   └─ ready
```

### 入力ロック

- `ready` と `selecting` 以外では盤面入力を受け付けない。
- 回転中の入力を予約しない。連打しても次の手へ持ち越さない。
- `pointercancel`、タブ非表示、ダイアログ表示時は選択中の交点を解除する。
- 非同期演出には解決IDを付け、リスタート後に古い処理が盤面を書き換えないようにする。

### タイマー

- `ready`、`selecting`、`rotating`、`validating`、`reverting` 中は進む。
- 成功手の `clearing`、特殊技、落下、補充、長いカットイン中は停止する。
- ポーズ、説明、設定、タブ非表示中は停止する。
- 不成立手の演出中もタイマーを進め、総当たり連打で時間を止められないようにする。
- 0秒になった時点で成立済みの手は最後まで解決する。
- 最後の解決で封印ゲージが100%に達した場合はクリアを優先する。

## 6. 盤面データモデル

```ts
type RotationBoard = BoardCell[][]; // 常に8×8
type RotationDirection = "clockwise" | "counterclockwise";
type RotationPoint = { row: number; column: number }; // 0〜6

type RotationMove = {
  point: RotationPoint;
  direction: RotationDirection;
};

type RotationPhase =
  | "idle"
  | "ready"
  | "selecting"
  | "rotating"
  | "validating"
  | "reverting"
  | "clearing"
  | "special-resolving"
  | "falling"
  | "refilling"
  | "shuffling"
  | "paused"
  | "clear"
  | "timeout";
```

- 盤面は各行・各列が常に8個であることを純粋関数の入口で検証する。
- 空マスは解決処理中だけ許可し、`ready` へ戻る前に必ず全64マスを補充する。
- UIは盤面を直接変更せず、純粋ロジックが返した新しい盤面と解決ステップを描画する。
- 乱数は関数引数から注入し、テストで同じ補充と連鎖を再現できるようにする。

## 7. 2×2回転

交点 `(row, column)` が対象とする4マス:

```text
A B
C D
```

- 時計回り: `C A / D B`
- 反時計回り: `B D / A C`
- 入力盤面は変更せず、新しい盤面を返す。
- 時計回り4回、反時計回り4回で完全に元へ戻ることをテストする。
- 不成立復帰では、見た目だけでなく通常色、特殊ブロック、妨害状態も操作前のスナップショットへ戻す。

### 成立手の条件

次のどちらかを満たせば成立手とする。

1. 回転後に通常色の4個以上一致が1組以上できる。
2. 回転によって同種特殊ブロックが上下左右に隣接し、スーパー効果が新たに成立する。

回転前から存在する一致やスーパー隣接は、安定盤面生成時に残さない。

## 8. 一致、報酬、特殊ブロック

### 通常一致

- 縦、横、右下がり斜め、右上がり斜めを走査する。
- 同じ直線を重複登録しない。
- 交差したマスは消去数として1回だけ数える。
- 1手目の直接消去を `1 CHAIN`、補充後の再消去を `2 CHAIN` 以降とする。
- 現行の連鎖倍率 `1 / 3 / 10 / 25 / 60 / 120` を維持する。

### 最新の報酬優先順位

1. 縦・横・斜めを同時に含み、重複を除いて9個以上消去: プリズムブレイク。
2. 縦5個以上: チェインピラー。
3. 横5個以上: チェインウェーブ。
4. 斜め5個以上: チェインボム。

- プリズムブレイク条件を満たした消去では、他の特殊ブロック報酬を生成しない。
- プリズムブレイク以外は、独立した長い直線が複数ある場合に各直線へ報酬を出せる。
- 報酬配置候補は「今回回転した4マス内の消去対象」を優先し、なければ該当直線の中央付近を選ぶ。
- 同じマスへ複数報酬を置かない。

### 特殊ブロック

| ブロック | 通常効果 | スーパー効果 |
| --- | --- | --- |
| チェインボム | 周囲3×3 | グランドチェインボム、周囲5×5 |
| チェインピラー | 縦1列 | トリニティピラー、縦3列 |
| チェインウェーブ | 横1列 | トリニティウェーブ、横3列 |
| プリズムブレイク | 接した消去色を全消去。効果巻き込み時はランダム色 | プリズムノヴァ、通常色を全消去 |

- 通常色の消去に上下左右で接した場合、または別の特殊技に巻き込まれた場合に発動する。
- 同種特殊ブロックが上下左右で隣接すると自動でスーパー効果になる。
- 斜め隣接だけではスーパー効果にしない。
- 補充で特殊ブロックが出ても、単独では発動しない。
- 特殊効果で発生した空マスも重力・補充対象となり、その後の一致を連鎖として扱う。

### 8×8盤面でのグランドチェインボム再調整

- 5×5消去は8×8盤面の最大約39%を一度に消すため、現状のままでは強すぎる可能性が高い。
- Phase R4の初期実装時は既存効果を仮接続して比較できるようにするが、正式値として固定しない。
- 優先する調整候補は、5×5から「中心3×3＋上下左右の先端4マス」へ変更する方式。
- 次点は5×5のまま対象数に上限を付ける方式、または盤面を8×10へ拡張する方式。
- 盤面行数は将来ステージ設定から変更できる構造を目標にする。ただし、R2の初回プレイ版は検証済みの8×8を維持する。
- 爽快感、通常ボムとの差、プリズムノヴァとの強弱関係を実機で比較してから確定する。

### 補充抽選の初期値

- 新規補充1マスごとの特殊ブロック合計率: 2.0%。
- チェインボム: 0.65%。
- チェインピラー: 0.55%。
- チェインウェーブ: 0.55%。
- プリズムブレイク: 0.25%。
- 盤面上の特殊ブロック上限: 6個。
- 20個連続で特殊ブロックが出なかった場合、次の補充時に通常3種のいずれかを保証する救済を候補とする。
- 大消去報酬は上限判定より優先するが、1回の解決終了時に上限超過を記録し、次のランダム補充を停止する。

数値はすべてステージ設定へ置き、実機確認後に調整する。

## 9. 初期盤面、補充、手詰まり

### 初期盤面生成

1. 4色で64マスを生成する。
2. 完成済みの4個以上一致がないことを確認する。
3. 時計回り49手、反時計回り49手を仮適用する。
4. 成立手が1件以上あることを確認する。
5. 条件を満たさなければ再生成する。

- 通常は最大200回まで再生成する。
- 上限到達時は、成立手を埋め込んだ安全なテンプレートを使う。
- 初期盤面には特殊ブロックと妨害状態を置かない。

### 補充

- 消去後、列ごとに既存ブロックを下へ詰める。
- 空いた個数だけ上端へ新しいブロックを生成する。
- 補充アニメーション用に、各ブロックの開始行・終了行を解決ステップへ含める。
- 補充後に一致またはスーパー隣接があれば、入力を戻さず自動解決する。
- 自動解決の連鎖には安全上限を設ける。初期値は30段階とし、到達時はログを残して安定盤面へ再構築する。

### 手詰まり

- 安定盤面ごとに98通りの回転を評価し、成立手一覧をキャッシュする。
- 0件なら「彩鎖を結び直しています」演出後に自動シャッフルする。
- シャッフル後も「完成済み一致なし」「成立手1件以上」を保証する。
- シャッフル中はタイマーを停止する。
- 10秒無操作でヒント候補を1つ発光する。設定で自動ヒントをOFFにできるよう拡張可能にする。

## 10. スコア、封印ゲージ、勝敗

### スコア

- 現行の通常消去点、ライン点、特殊技点、連鎖倍率を基準にする。
- 不成立手は0点。
- シャッフルは減点なし。
- 統計としてスコア、最大チェイン、消去数、成功手数、不成立手数、残り時間を保持する。

### 封印ゲージ

- 表示は0%から100%へ増える「封印ゲージ」とする。
- 通常消去、連鎖、特殊技の結果を封印値へ変換する。
- 初期調整値は「平均的なプレイで60〜80秒程度で100%」を目標とする。
- 最初の仮式:
  - 通常消去1個: 0.45。
  - 2 CHAIN以降: 連鎖段階ごとに追加倍率。
  - 軽被弾技: 通常消去分に追加3。
  - 中被弾技: 追加8。
  - 大被弾技: 追加18。
- 封印値の式は純粋関数へ分離し、スコアとは独立して調整する。

### 勝敗

- 90秒以内に封印ゲージ100%でステージクリア。
- 0秒で100%未満ならゲームオーバー。
- 0秒直前の成功手は最後まで解決する。
- 最終解決で100%へ到達すればクリア。
- クリア後は結果画面でスコア、最大チェイン、残り時間、成功率を表示する。

### 後半ステージ用の目標バリエーション

- 制限時間式に加えて「規定手数以内に指定数を消す」手数制ステージを追加できる構造にする。
- 手数は成立・不成立を問わず、盤面を実際に回転した時点で1手として数える案を初期候補とする。
- 通常消去では壊れないターゲットブロックを盤面へ置き、消去・重力・補充を利用して最下段まで落とすステージを追加候補とする。
- ターゲットは通常色の一致判定へ参加せず、最下段到達後に盤面外へ回収された時点で目標数を加算する。
- ステージ目標は `time-limit`、`move-limit`、`drop-target` を差し替えられる設定データとして保持し、盤面解決ロジックへ個別条件を直書きしない。
- 初期のモコスライム戦と基本操作が安定してから導入し、1ステージへ複数の新ルールを同時投入しない。

## 11. 補助技の回転式仕様

### タイムヴェール

- 現行スロータイムの代替。
- ブロック消去でゲージをためる。
- 発動後6秒間、プレイヤーが操作中でも制限時間だけを停止する。
- 敵妨害ゲージは通常どおり進めるか、初回試作では同時停止とし、実機で強さを確認する。
- チェイン解決中はもともと時間が止まるため、効果時間を消費しない。

### チェインウェーブ

- 現行の横レーザー補助技を維持する。
- 消去数でゲージをため、準備完了後に行を選択して横1列を消す。
- 行選択中も制限時間は進む。
- キャンセル可能。
- 消去後は重力、補充、連鎖判定を行う。

### HOLD

- 落下ペアが存在しないため廃止する。
- HUDの空いた位置はヒントボタンとステージ目標表示へ置き換える。

## 12. モコスライムの妨害

### 回転式「ぬめり結び」

- 妨害ゲージが満タンになる前に対象交点を予告する。
- 発動すると1つの交点へぬめりを付け、6秒間その交点を回転不可にする。
- 見た目は半透明の緑色膜、鎖、禁止記号で示し、色だけに依存しない。
- 盤面のブロック色や特殊ブロックを隠さない。
- 封鎖中の交点をタップした場合は短い揺れと説明を出し、盤面は変更しない。

### 公平性

- 妨害対象を決める前に成立手一覧を確認する。
- その交点を封じても成立手が1つ以上残る場合だけ採用する。
- 安全な交点がなければ妨害を不発にし、ゲージを一定量戻して再充填する。
- 同時に封じる交点は初期ステージでは1つだけ。
- チェイン消去で敵妨害ゲージを遅らせる現行思想を維持する。
- 特殊技または3 CHAIN以上で、ぬめり残り時間を短縮する救済を候補とする。

## 13. 入力仕様

### タッチ・マウス

- 盤面内側の交点49個に、セル寸法から算出した判定領域を置く。
- 判定領域は隣接交点と重ならない。
- タップ許容移動量: 12px。
- タップ最長時間: 320ms。
- スワイプ成立距離: `min(24px, セル幅×0.55)` を初期値とする。
- 横移動が縦移動の1.2倍以上なら左右スワイプとして扱う。
- 縦方向優勢、複数指、右クリック、非主ポインターはキャンセルする。
- 盤面だけ `touch-action: none` とし、説明画面などのスクロールを妨げない。

### 視覚フィードバック

- 押下時に対象2×2を太枠で囲む。
- 時計回り候補は右回り矢印、反時計回り候補は左回り矢印を表示する。
- スワイプ中は距離に応じて矢印を強調する。
- 不成立時は対象2×2を短く橙色にしてから逆回転する。
- 成立時は対象枠を鎖色に発光させ、最初の消去位置へCHAINテロップを出す。

### キーボード・支援技術

- 交点を49個の操作ボタンとして扱う。
- 矢印キーで交点フォーカスを移動。
- Enter／Spaceで時計回り。
- Shift+EnterまたはZで反時計回り。
- 各交点へ「上から3、左から4の交点、時計回り」などの日英ARIAラベルを設定する。
- 回転結果、CHAIN、特殊技、手詰まり、残り10秒、勝敗を `aria-live` で簡潔に通知する。
- 色覚補助記号を通常ブロックに残す。

## 14. 横長固定UI

### 固定論理画面

```text
┌─────────┬────────────────┬────────────┬─────────┐
│ クロマ   │    8×8盤面      │ 時間・スコア │ モコ     │

```

- 現在の木製フレーム、暖色パネル、固定背景を継承する。
- 8×16から8×8になるため、盤面セルを大きくする。
- NEXT、HOLD、落下操作ボタンを撤去する。
- HUDは「残り時間、スコア、封印ゲージ、最大CHAIN、ヒント、補助技」に置き換える。
- 説明・設定ボタンは安全領域内の右上へ維持する。
- 論理解像度は暫定で1280×720とし、内部の位置・寸法はこの座標系だけで設計する。
- 実画面では1280×720全体をアスペクト比を維持したまま拡大・縮小し、UI部品の並び替えは行わない。
- 16:9より横長の端末は左右、縦長の端末は上下に余白を設ける。背景は余白まで延長できるが、操作UIは固定画面内に収める。
- ノッチとシステムバーの安全領域は、外側ラッパーの余白として処理する。固定画面内部の配置は変えない。
- ゲーム進行中は縦・横スクロールを発生させない。
- PCのマウス座標とスマートフォンのポインター座標は、表示倍率を除いて論理座標へ変換してから交点判定へ渡す。

### 縦持ち時

- ブラウザでは画面方向を確実に強制できないため、固定UIを縦向けへ組み替えない。
- ゲーム開始前に横向き案内を表示し、横向きになったら同じ1280×720画面を拡大して表示する。
- 縦持ちでも開始を禁止はしないが、操作性検証の対象外とする。
- 将来のスマートフォンアプリでは横画面固定をネイティブ設定で指定する。

### ブラウザからアプリへの移植境界

- ゲーム本体、盤面解決、React UI、演出、日英テキスト、素材管理はそのまま共有する。
- 画面方向固定、ステータスバー、戻る操作、端末ストレージ、振動、音声フォーカスだけを環境アダプターへ分離する。
- 最終候補はWebView方式を優先し、Capacitor等の採用可否はWeb版完成後にAndroid実機ビルドで判断する。
- ブラウザ固有APIをゲームロジックから直接呼ばず、将来差し替え可能な薄い関数へ集約する。

## 15. ルーティングと公開分離

現在の `App.tsx` はテストURLを1件だけ個別判定しているため、回転式追加時にテストルート定義を小さな設定表へ整理する。

```ts
const testGameRoutes = {
  "/test/color-chain-mascot/": { id: "colorChainMascotTest", ... },
  "/test/color-chain-rotate/": { id: "colorChainRotationTest", ... }
};
```

- 回転式は遅延読込する。
- `<main>` へ回転式専用クラスを付ける。
- title、description、canonical、OG情報を回転式用に設定する。
- robotsは `noindex,nofollow`。
- ポータル一覧、お気に入り、最近遊んだ、人気、新着には追加しない。
- サイトマップへ追加しない。
- PWAの `navigateFallback: /index.html` は `/test/` を除外していないため、基本設定の変更は不要。
- Cloudflare Pagesで直URLを開き、リロードしても同じ試作ページへ戻ることを確認する。

- 初回説明表示: `game-shelf-color-chain-rotate-v1-tutorial`。


### 共通化の順序

1. 色・特殊ブロックの定数と型だけを `tokens.ts` へ移す。
2. `logic.ts` から再エクスポートし、現行テストが無変更で通ることを確認する。
3. 回転式は `tokens.ts` を参照する。
4. 一致・特殊解決の共通化は、回転式が安定してから検討する。

この順序により、最初から現行落下式の盤面処理を大きく移動しない。

## 19. 自動テスト項目

### 回転

- 時計回り・反時計回りの4マス移動。
- 同方向4回で原状復帰。
- 時計回り1回＋反時計回り1回で原状復帰。
- 盤面端に存在しない交点を拒否。
- 入力盤面を変更しない。

### 一致と連鎖

- 縦、横、右下がり、右上がり。
- 4個、5個、6個以上。
- 交差、独立した複数同時消し。
- 斜め5個以上のチェインボム。
- 縦・横・斜めを含む9個以上同時消しのプリズムブレイク単独報酬。
- 重力後の2 CHAIN、補充後の3 CHAIN以上。
- 30段階安全上限。

### 盤面保証

- 初期一致なし。
- 成立手1件以上。
- シャッフル後も同じ保証。
- 4色、将来の5色双方。
- 1000シード以上の反復。

### 特殊ブロック

- 4種の通常効果。
- 4種のスーパー効果。
- 上下左右隣接と斜め非発動。
- 効果巻き込みの連鎖。
- プリズムブレイクの隣接色／ランダム色。
- 報酬配置競合。
- 特殊ブロック上限とレア率。

### 時間と勝敗

- 不成立復帰中は時間が進む。
- 成功手の解決中は時間が止まる。
- ポーズ・説明・設定・非表示中は停止。
- 0秒直前の成立手を最後まで解決。
- 最終解決のクリア優先。

### 入力

- タップ、左右スワイプ、縦キャンセル。
- 複数指、pointer cancel、画面外移動。
- 解決中の二重入力拒否。
- キーボード交点移動と両方向回転。

# カラーチェイン 横長プレイ画面・ステージ演出仕様 v1

## 文書の位置付け

- 状態: 将来実装向けの構成案
- 対象: `カラーチェイン`、公開テスト版 `クロマのマジカルチェイン`、将来のステージ制・敵戦
- 方針: 先に敵キャラクターと各ステージの背景を固め、画面の全面刷新はその後に着手する
- 現在の盤面ロジック、公開URL、操作方式はこの時点では変更しない

## 目標

- PCでは横長のゲーム画面を基本に、盤面・クロマ・敵・補助技を一画面へ収める
- スマートフォンでは横画面を推奨し、タップとスワイプで直感的に操作する
- カテナリアの「鎖・宝石・魔法」の世界観を、ステージ背景、外枠、UI、ブロック、演出へ一貫して反映する
- 背景やキャラクターが豪華になっても、落下ブロックと次ブロックの視認性を最優先する

## 画面構成

```text
┌──────────────────────────────────────────────┐
│ SCORE / MAX CHAIN / STAGE / PAUSE / SETTINGS  │
├───────────┬──────────────────┬───────────────┤
│ クロマ枠    │     8 × 16 盤面    │ 敵キャラ枠      │
│ 表情・技名  │  CHAINテロップ      │ 封印/乱れゲージ │
│ モーション  │                    │ 妨害予告        │
├───────────┴──────────────────┴───────────────┤
│ NEXT / HOLD      補助技アイコン＋ゲージ       HELP │
└──────────────────────────────────────────────┘
```

- PC・タブレット横画面: クロマ、盤面、敵を3列構成にする
- スマートフォン横画面: 盤面を中央に固定し、クロマ・敵は細い縦枠または胸上表示にする
- スマートフォン縦画面: 横向き推奨の案内を出す。ただし、遊べない状態にはせず、盤面中心の簡易配置へフォールバックする
- 大技やカットインは盤面操作を遮らない短時間表示を基本とする

## 画面方向の扱い

- 通常のモバイルブラウザでは、横画面の強制固定を前提にしない
- 全画面化後に `ScreenOrientation.lock("landscape")` を試行し、失敗した場合は横画面推奨表示だけを行う
- インストール済みPWAでは、Manifestの `orientation: "landscape"` を将来の候補とする
- Android／Steamなどのネイティブアプリ版では、横画面固定を正式に採用できる
- 縦画面では常に操作可能な代替レイアウトを残す

## スマートフォン操作仕様

盤面内だけをジェスチャー対象とし、ブラウザの戻る操作やページスクロールと競合しないようにする。キーボード操作と画面ボタンは代替操作として残す。

| 入力 | 動作 | 誤操作対策 |
| --- | --- | --- |
| 短いタップ | 右回転 | 指の移動量が10〜14px未満の場合だけ判定 |
| 横スワイプ | 左右移動 | 1マス相当の距離ごとに1回移動。開始位置が画面端の場合は無効化候補 |
| ゆっくり下スワイプ | ソフトドロップ | 下方向の距離に応じて段階的に落下 |
| 素早い下フリック | ハードドロップ | 最低移動距離と下方向速度の両方を満たした時だけ発動 |
| 補助技アイコンのタップ | 技を発動 | 対象選択技は発動後に盤面を1回タップして確定 |

- `touch-action: none` は盤面にだけ適用する
- 初回は「ジェスチャー＋画面ボタン併用」を標準とし、プレイ検証後に設定で切替可能にする
- ゲームパッド、キーボード、アクセシビリティ向けの画面ボタンは維持する

## ステージテーマ

各ステージを `StageTheme` 相当のデータとして管理し、画面構造を変えずに演出だけ差し替えられるようにする。

| テーマ要素 | 内容 |
| --- | --- |
| 背景 | 遠景・中景・前景の2〜3層。静止画を基本とし、軽い粒子だけを重ねる |
| 盤面外枠 | 木目、レンガ、石造、氷、古書、鏡など、地域に対応した意匠 |
| クロマ枠・敵枠 | 背景と同系統だが、盤面より彩度を抑えてキャラクターを見やすくする |
| UIアクセント | ボタン、ゲージ、CHAIN文字の色、鎖の発光色 |
| BGM | ステージごとのループ曲。開始操作後のみ再生し、音量設定・無音設定を用意する |
| 効果音 | 消去、CHAIN、特殊技、敵予告、勝敗。BGMと別音量で管理する |

- 盤面内部は暗めの半透明レイヤーを維持し、背景の派手さで色ブロックを見失わないようにする
- 動画背景や常時Canvas描画は初期採用せず、低性能端末でも60fpsを保ちやすい静止画＋CSS演出を優先する
- BGM・SEは将来の外部素材または生成音源を採用する。配布・商用利用・再配布条件を確認し、素材名、作者、ライセンス、取得元を台帳で管理する

## ブロックの新デザイン

- 基準資料: `reference/color-chain-block-style-v1.png`
- 初級標準の4色は、青、ピンク、黄、紫を基本候補とする
- 宝石のような明るい外縁、厚み、中央面、左上の光沢、下側の陰影を共通構造にする
- 特殊ブロックは、通常ブロックの宝石構造を保ったまま、鎖・鏡・魔法陣を重ねて区別する
- 色だけに依存せず、淡いルーン、形状記号、またはアイコンを併用して色覚特性への配慮を残す
- 実装はセルごとのラスター画像貼り付けではなく、CSSグラデーション・擬似要素・必要最小限のSVGで再現する。拡大縮小と描画負荷を安定させるためである

## CHAINテロップ

消去したブロック群の中心付近に、短時間で大きく表示する。

| 発生条件 | 表示案 |
| --- | --- |
| 1連鎖 | `1 CHAIN!` |
| 2連鎖 | `2 CHAIN!` |
| 3連鎖 | `3 CHAIN!` |
| 4連鎖以上 | `HYPER 4 CHAIN!` など、段階名も併記 |
| 特殊技と重複 | 上段に技名、下段にCHAIN数を表示 |

- 斜体、太いアウトライン、鎖を思わせる縁取り、段階ごとの虹色または宝石色を使う
- 600〜900ms程度で拡大、軽い跳ね、残光、消散を行う
- 表示座標は盤面内へ補正し、端で切れないようにする
- 落下中の操作ブロックを覆いすぎないよう、短時間・半透明・表示数制限を設ける
- 視覚演出を減らす設定では静的で読みやすい小さな表示へ切り替える

## 説明・設定画面

- 現在の長い説明エリアは、ゲーム内モーダルまたはドロワーへ移す
- PCでは右側ドロワー、スマートフォンでは全画面シートを基本とする
- 開いている間はゲームを自動ポーズし、閉じる時に再開する
- タブは「遊び方」「特殊ブロック」「補助技」「操作」「設定」に分ける
- 設定にはBGM、SE、画面効果、キャラクター表示、操作方式、ジェスチャー感度、色覚補助を置く

## 補助技HUD

- 補助技は「アイコン＋充填ゲージ＋使用可能時の発光」へ統一する
- タップまたはクリックで即時発動し、対象選択が必要な技だけ盤面選択モードへ入る
- 使えない時は暗く表示し、使用可能時は金色または虹色の縁を出す
- アイコンの詳細説明はHUD内ではなく、説明画面に集約する
- 各アイコンはスマートフォンで最低44px程度のタップ領域を確保する

## 実装順

1. 横長バトル用の画面構造を新設し、縦画面フォールバックを用意する
2. タップ・スワイプを入力コントローラーとして分離する
3. 説明・設定をモーダル化し、補助技をアイコンHUDへ統合する
4. 新ブロックデザインとCHAINテロップを導入する
5. `StageTheme` による背景・枠・UIアクセントの差し替えを実装する
6. BGM・SE・音量設定を追加する
7. クロマ、敵キャラ、封印／乱れゲージ、妨害予告を接続する
8. ステージごとのカットイン、モーション、画面効果を増やす

## 実装前に必要な素材と判断

- 各ステージの背景: PC向け横長、スマートフォン横長、必要なら縦画面向けのトリミング方針
- 盤面外枠・クロマ枠・敵枠: 9スライスまたは繰り返し可能な木目／レンガなどの素材
- BGM・SE: 利用条件が確認できる音源、または生成音源の利用規約・制作記録
- UIアイコン: 補助技、設定、音量、説明、ポーズ
- フォント: 商用Web配信可能で、日英両方を読みやすく表示できるもの
- 実機QA: iOS Safari、Android Chrome、PCブラウザ、低性能端末での入力・描画・音量確認

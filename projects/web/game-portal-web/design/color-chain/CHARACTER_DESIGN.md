## 世界観と用語

同色ブロックを縦・横・斜めに4個以上そろえて消す現象を、魔法の鎖が結ばれる「マジカルチェイン」と呼ぶ。連続消去は次の技名で統一する。

| 段階 | 日本語 | 英語 |
| --- | --- | --- |
| 1 | マジカルチェイン | Magical Chain |
| 2 | Wチェイン | Double Chain |
| 3 | トリプルチェイン | Triple Chain |
| 4 | ハイパーマジカルチェイン | Hyper Magical Chain |
| 5 | ギガマジカルチェイン | Giga Magical Chain |
| 6以上 | アルティメットマジカルチェイン | Ultimate Magical Chain |

- クロマはブロックを直接破壊するのではなく、同じ色の魔力を鎖で結んで解放する
- 連鎖数が増えるほど、細い光の輪から太く複雑な魔法鎖へ成長する
- 爆弾、レーザー、カラーブレイカーの魔法陣にも、円環ではなく連結した鎖の意匠を入れる
- 最上位演出ではマゼンタ、シアン、ゴールドの三色の鎖をクロマの杖へ集束させる

### 特殊魔法名（仮設定）

| 効果 | 通常技 | 強化技 |
| --- | --- | --- |
| 3×3爆発 | チェインボム / Chain Bomb | グランドチェインボム / Grand Chain Bomb |
| 縦1列 | チェインピラー / Chain Pillar | トリニティピラー / Trinity Pillar（縦3列） |
| 横1行 | チェインウェーブ / Chain Wave | トリニティウェーブ / Trinity Wave（横3行） |
| 同色全消去 | プリズムブレイク / Prism Break | プリズムノヴァ / Prism Nova（全色） |

- 内部実装のブロックIDは互換性のため `bomb`、`vertical-laser`、`horizontal-laser` のまま維持する
- 機能説明では初見でも方向が分かるよう「縦1列」「横1行」を併記する

## 変更してはいけない識別要素

- 紫紺の中程度の長さの髪。毛先の一部にマゼンタとシアンの差し色
- 紫の瞳と、明るく自信のある顔立ち
- 大きな濃紺の三角帽子。つばの裏は左側がマゼンタ、右側がシアン
- 帽子の金色チェーンと菱形バックル、つばに並ぶマゼンタ・紫・シアンの菱形模様
- 濃紺の短いケープとワンピース。金色の縁取りとチェーン装飾
- スカート内側はマゼンタとシアンに分かれる
- 左右でマゼンタとシアンを分けた靴底の差し色
- 金色の鎖を軸にした短い杖と、先端の虹色プリズム結晶
- 杖の鎖はゲーム内の魔法鎖と同一意匠にし、クロマがチェイン魔法の使い手だと分かるようにする
- 小柄で機敏だが、幼児体型にはしない

## 生成時の共通指定

- 基準画像を参照画像として必ず入力する
- 同じ顔、髪型、帽子、衣装、杖、配色を維持する
- 既存作品のキャラクターや画風を指定しない
- 装飾を追加しない。特にフリル、宝石、ベルト、鎖を勝手に増やさない
- 左右のマゼンタ／シアン配置を全資料で固定する
- 杖の長さと結晶形状を固定する
- セル塗り寄りのゲーム用イラスト。細密な厚塗りにしない
- 透過素材化を見越し、輪郭を明瞭にし、髪や衣装を背景と混色させない
- 資料画像には文字、ロゴ、透かしを入れない

## 制作順序

### 1. 三面図

- 正面、真横、背面
- ニュートラルな直立姿勢
- 帽子あり／杖あり
- 表情は同一の自然な微笑み
- 正投影に近い見え方で、極端な遠近を付けない
- 衣装の前後関係、チェーン、ケープ、スカートの構造確認を目的とする

### 2. 表情シート

- 通常
- 自信あり
- 小さな成功
- 大喜び
- 驚き
- 焦り
- 泣きそう
- ゲームオーバーで落ち込む
- 顔の角度、髪、帽子の位置をそろえる

### 3. デフォルメ版

- 2.5～3頭身
- 通常版と同じ帽子、髪、衣装、杖
- PCでは全身、スマートフォンでは胸上表示でも判別可能なシルエット
- 待機、喜び、焦り、敗北、大技の5ポーズ

### 4. モーション原画

- 待機: 盤面を見る、瞬き、杖を持ち直す
- マジカルチェイン: 杖で小さな光の鎖を結ぶ
- Wチェイン: 小さく拳を握り、二重の鎖を引き寄せる
- トリプルチェイン: 杖を前へ出し、三本の鎖を束ねる
- ハイパーマジカルチェイン: 杖を掲げ、背景を横切る大きな魔法鎖を展開する
- ギガマジカルチェイン: 全身を使って巨大な鎖の魔法陣を解放する
- アルティメットマジカルチェイン: 虹色プリズムへ全色の鎖を集束し、決めポーズを取る
- 特殊ブロック: チェインボム、チェインピラー、チェインウェーブ、プリズムブレイク
- 危険: 盤面上部を見て身を乗り出す
- 敗北: 肩を落としてしゃがむ
- 各動作は開始、溜め、最大姿勢、戻りの最低4キーを用意する

### 5. ドット絵版

- 最初は `96 × 96 px` 相当の設計で作る
- 仕上げ時に手作業でピクセルグリッドへ整える
- 生成画像をそのまま縮小せず、帽子、顔、杖、ケープを優先して再構成する
- 32px級では全身ではなく胸上アイコンも別途用意する
- パレットは透過色を除いて24～32色程度を目安にする

### 6. 小物・演出資料

- 帽子単体の正面、横、背面
- 杖単体の正面と横
- 通常発光、チェインボム、チェインピラー、チェインウェーブ、プリズムブレイクの杖発光差分
- 小型魔法陣と大技用魔法陣
- 通常、W、トリプル、ハイパー、ギガ、アルティメットの魔法鎖
- 鎖の一つの輪、連結状態、切断・消散状態の設定画
- マゼンタ、シアン、ゴールドのクリスタルブロック

## 生成バッチの分け方

同一画像へすべて詰め込まず、次の単位で生成する。

1. 三面図のみ
2. 表情のみ
3. 通常等身のモーション原画
4. デフォルメ版の設定画
5. デフォルメ版のモーション原画
6. ドット絵の方向性サンプル
7. 杖・帽子・魔法陣・ブロックの小物シート

各バッチで採用画像を1枚決め、その採用画像を次の生成にも参照として加える。これにより世代をまたいだ外見の変化を抑える。

## 実装用画像に進む前の確認項目

- キャラクター名とゲーム名が確定している
- 通常等身かデフォルメ版のどちらをゲーム内表示に使うか決まっている
- PCとスマートフォンでの表示サイズが決まっている
- ボイスの有無にかかわらず口パクを用意するか決まっている
- 左右反転して使える動作と、反転禁止の動作を分類している
- 最終画像の権利・公開範囲を確認している

## テスト版での運用

- 本番版とは別URLのテスト画面 `/test/color-chain-mascot/` で確認する
- 現在はURLを知っている人が直接アクセスできる公開テストとして運用する
- 本番版とゲームロジックは共有し、キャラクターUIだけを分岐する
- テスト版は `noindex`、Analytics除外、別LocalStorageキー、別Service Workerスコープとする
- キャラクター表示、アニメーション、画面効果、ボイスを個別に無効化できる設計にする
